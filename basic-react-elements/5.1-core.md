### Node materials & TSL

Threefy 2.0 runs on the three.js WebGPU renderer, so custom shading is authored with **node materials** and **TSL** (Three.js Shading Language) instead of GLSL string shaders. Each node material accepts TSL nodes directly as props — `colorNode`, `positionNode`, `normalNode`, `emissiveNode`, `fragmentNode`, and so on — so your shading logic lives inside the component tree. TSL compiles to WGSL on WebGPU and to GLSL on the WebGL 2 fallback, so a single shader runs on both backends.

```jsx
import { ThreeCanvas } from 'threefy'
import { positionLocal, sin, time, vec3 } from 'three/tsl'

const PulsingBox = () => (
  <mesh>
    <boxGeometry args={[ 20, 20, 20 ]}/>
    <meshStandardNodeMaterial
      colorNode={vec3( sin( time ).mul(0.5).add(0.5), 0.4, 1 )}
      positionNode={positionLocal.mul( sin( time ).mul(0.05).add(1) )}
    />
  </mesh>
)
```

The `time` node advances automatically each frame, so time-based animation needs no `useFrame` uniform bookkeeping. Node materials can also be created through the short-form `<material type={'standardNode'} … />` (see [5.3 Extensions](5.3-extensions.md)).
