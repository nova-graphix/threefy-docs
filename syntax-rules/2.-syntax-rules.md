# 2. Syntax Rules

## 2.1 args

The following code is a JavaScript code that creates a box geometry using its constructor provided by three.js

```javascript
const boxGeometry = new THREE.BoxGeometry( width, height, depth, widthSegments, heightSegments, depthSegments )
```

The code below is a short version of the three.js code above using threefy.

```markup
<boxGeometry args={[ width, height, depth, widthSegments, heightSegments, depthSegments ]} />
```

Note that as in the example above, all input arguments to the class constructor found in three.js can be entered to the args property of the element provided by threefy. The following table summarizes these features of threefy. The left and right code snippets below produce identical results.

| three.js                                  | threefy                    |
| ----------------------------------------- | -------------------------- |
| const mesh = new THREE.Mesh( …arguments ) | \<mesh args={arguments} /> |

## 2.2 property (eg: position, rotation, scale)

The following code defines the position, scale, and rotation of the mesh in three.js.

```js
const mesh = new THREE.Mesh(...)
mesh.position.set( px, py, pz )
mesh.scale.set( sx, sy, sz )
mesh.rotation.set( rx, ry, rz )
```

The above code can be rewritten as shown below.

```markup
<mesh position={[px,py,pz]} scale={[sx,sy,sz]} rotation={[rx,ry,rz]} />
```

In the above code, px, py, and pz represent the x, y, and z coordinates, respectively; sx, sy, and sz represent scaling factors in the x, y, and z directions, respectively; and rx, ry, and rz represent the rotation angles (in radians) in the x, y, and z directions, respectively. The comparison table below summarizes these features of threefy. The left and right sides of the table represent three.js code and threefy code, respectively, and produce the same result.

| three.js                                                         | threefy                    |
| ---------------------------------------------------------------- | -------------------------- |
| <p>const mesh = new THREE.Mesh()</p><p>mesh.property = value</p> | \<mesh property={value} /> |

## 2.3 method (as like a property)

In three.js, a 3D object with a method executes its function by inputting a value (or function) to that method and then calling the method. In threefy, an element with a method treats the method as a property, setting a value (or function) to the method property to execute the method. The following two examples illustrate these features, and the last example summarizes them.

<table><thead><tr><th width="401">three.js</th><th>threefy</th></tr></thead><tbody><tr><td><p>const material = new THREE.MeshBasicMaterial()</p><p>material.onBeforeRender = (renderer, scene, camera) => { ... }</p></td><td><p>&#x3C;meshBasicMaterial</p><p>  onBeforeRender={(renderer, scene, camera) => { ... }}</p><p>/></p></td></tr><tr><td><p>const mesh = new THREE.Mesh()</p><p>mesh.rotateY( Math.PI/4 )</p><p>mesh.translateX( 10 )</p></td><td><p>&#x3C;mesh</p><p>  rotateY={Math.PI/4}</p><p>  translateX={10}</p><p>/></p></td></tr><tr><td><p>const object = new THREE.Object3D()</p><p>object.method( value )</p></td><td>&#x3C;object3D method={value} /></td></tr></tbody></table>

In the table above, onBeforeRender() is a method of the THREE.MeshBasicMaterial object, while rotateY() and translateX() are methods of the THREE.Mesh object. For more information, see [https://threejs.org/docs/](https://threejs.org/docs/).

> **Note (v2.0 / WebGPU):** the GLSL-based `onBeforeCompile` hook is not available under the WebGPU renderer. To customize shading, use **node materials** with **TSL** (see [5.1 Core](../basic-react-elements/5.1-core.md) and [5.3 Extensions](../basic-react-elements/5.3-extensions.md)) instead of patching compiled GLSL.

## 2.4 attach

The following code shows how to use _attach_. The word, attach is a built-in keyword that does not indicate a child relationship of a parent element, but indicates that it is a property of the parent element.

```markup
<mesh scale={5}>
    <boxGeometry args={[10,10,10]} attach='geometry' />
    <meshBasicMaterial color={color} attach='material' />
</mesh>
```

In the above code, we have that mesh.geometry = boxGeometry and mesh.material = meshBasicMaterial. Note that attach can be omitted in case of attach='geometry' or attach='material' for readability of the code.

## 2.5 dash(-)

The position.x in three.js is expressed as the position-x in threefy. Similarly, scale.y and rotation.z are expressed as scale-y and rotation-z, respectively.

```javascript
mesh.position.x = 10
mesh.scale.y = 2
mesh.rotation.z = Math.PI/2
```

```markup
<mesh position-x={10} scale-y={2} rotation-z={Math.PI/2} />
```

## 2.6 color

Colors in three.js can be expressed in various ways as follows.

```javascript
<material type={'basic'} color={'red'} />
<material type={'basic'} color={0xff0000} />
<material type={'basic'} color={[1,0,0]} />
<material type={'basic'} color={new THREE.Color(1,0,0)} />
```

## 2.7 texture url

The texture of three.js can be created using the texture loader function of three.js as follows, but it can also be created by simply entering the url.

```markup
<meshBasicMaterial map={new THREE.TextureLoader().load(url)} />
<meshBasicMaterial map={url} />
```

## 2.8 texture sub-properties

Once a texture map is set (by URL or `THREE.Texture`), its sub-properties can be adjusted declaratively as props, without reaching for a ref. There are two forms.

**Per-map** — `<mapName>-<sub>`, where `mapName` is any texture map on the material (`map`, `normalMap`, `roughnessMap`, `aoMap`, `emissiveMap`, …):

```markup
<meshStandardMaterial
    map={url}
    map-repeat={[2,2]}          // [u,v]; a scalar (e.g. map-repeat={2}) means uniform [v,v]
    map-offset={[0.1,0]}
    map-rotation={0.5}          // radians
    map-anisotropy={16}
    map-wrap={'repeat'}         // 'repeat' | 'clamp' | 'mirror' (sets both wrapS and wrapT)
    normalMap-wrapS={'mirror'}  // wrapS / wrapT can also be set individually
    map-colorSpace={'srgb'}     // 'srgb' | 'linear' | 'none'
/>
```

**All maps** — `texture<Sub>` applies the same sub-property to every loaded texture map on the material at once:

```markup
<meshStandardMaterial
    map={colorUrl} normalMap={normalUrl} roughnessMap={roughUrl}
    textureRepeat={2}            // repeat all maps 2×2
    textureWrap={'repeat'}
    textureAnisotropy={16}
    textureColorSpace={'srgb'}
/>
```

Notes: setting `repeat` automatically switches the wrap mode to `RepeatWrapping` when it was still the default `ClampToEdge` (otherwise tiling would be invisible). Sub-props are applied after the base map has loaded, and threefy clones the texture on first sub-prop write so that other users of the same cached URL are not affected.
