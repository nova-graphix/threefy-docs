# 6.3 Element to support a ray marching algorithm

\<Raymarch /> is a special element that creates and renders a signed distance function using the ray marching algorithm, similar to [shadertoy](https://www.shadertoy.com/).\
\*NOTE: This element is unfortunately not included in the current version as it requires a number of shader sources and image files. It will be released as an addon to threefy soon.

> **Note (v2.0 / WebGPU):** since threefy 2.0 runs on the WebGPU renderer, the ray-marching shader is authored in **TSL** (Three.js Shading Language) rather than GLSL strings. The raymarching loop is expressed as a fragment node (`fragmentNode`) attached to a full-screen node material, which compiles to WGSL on WebGPU and to GLSL on the WebGL 2 fallback.

```jsx
<Raymarch
    fragmentNode={sdfNode}                    // TSL fragment node defining the SDF & march loop
    uniforms={uniforms}                       // values bound as TSL uniform() nodes
    textures={[ textureURL1, textureURL2, … ]}
/>
```

Here `fragmentNode` is a TSL node graph. If not set, a default node that outputs a transparent color is used.
