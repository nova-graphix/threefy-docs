# 7.6 Models & assets

This group is about bringing external content into a scene — loading 3D models, cloning them, building meshes from plain data, instancing a model thousands of times, animating sprites, playing spatial audio, and round-tripping a model back out to a file. The `<Model>` element and the `useThree` `animator` do most of the work. See also [6.1 Element to create or load 3D models](../advanced-react-elements/6.1-element-to-create-or-load-3d-models.md).

## Loading and cloning a model [(demo 31)](7.8-demo-sources.md#demomodelloading)

`<Model url={…}>` loads `.glb` / `.gltf` / `.zip` / `.vox` and hands you the object in `onLoad`. To place many copies, clone it with `SkeletonUtils.clone` (which preserves skinned-mesh bones) and drop each clone into a `<primitive>`.

```jsx
const [model, setModel] = useState(null);

if (!model) {
  return <Model url={'models/jump-animation.glb'} onLoad={setModel} />;
}

// once loaded, detach the original and scatter clones over a grid
model.parent?.remove(model);

return grid3D(-x0, x0, dx, -x0, x0, dx, -x0, x0, dx).map((p, i) => (
  <primitive key={i} object={SkeletonUtils.clone(model)} position={p} />
));
```

`<primitive object={…}>` wraps any existing three.js object as an element. Cloning is far cheaper than loading the file repeatedly — load once, clone N times.

## Creating models from data [(demo 32)](7.8-demo-sources.md#demomodelcreation)

`<Model>` can also **build** an object from plain descriptors instead of loading a file — pass `object`, `geometry`, and `material` as data. This keeps geometry/material construction declarative.

```jsx
// a textured box, built from descriptors
<Model
  object={{ type: 'Mesh', name: 'box' }}
  geometry={{ type: 'BoxGeometry', parameters: [20, 20, 20] }}
  material={{ type: 'MeshStandardMaterial', map: 'images/crate.gif' }}
/>

// a physical clearcoat sphere
<Model
  object={{ type: 'Mesh', name: 'sphere' }}
  geometry={{ type: 'SphereGeometry', parameters: [20, 64, 64] }}
  material={{ type: 'MeshPhysicalMaterial', metalness: 0.9, roughness: 0.5,
             clearcoat: 1.0, clearcoatRoughness: 0.1, color: 0xff0000 }}
/>

// a point cloud (v2 renders Points via instanced sprites — no uv crash, no 1px dots)
<Model
  object={{ type: 'Points' }}
  geometry={{ type: 'BufferGeometry', attributes: { position: positions, color: colors } }}
  material={{ type: 'PointsMaterial', map: 'images/sprites/metalBall.png', size: 3.0 }}
/>
```

The `type` strings map to three.js constructors and `parameters` are the constructor args. In v2, `Points` objects are delegated to an instanced-sprite renderer under the hood, which sidesteps WebGPU's point-rendering limitations.

## Instancing a whole model [(demo 39)](7.8-demo-sources.md#demoinstancedmodel)

`<InstancedModel>` combines model loading with instancing: one model, thousands of animated copies, each with its own transform and color driven by callbacks.

```jsx
<InstancedModel
  url={'models/LeePerrySmith.glb'}
  count={count}
  scale={4}
  animIdx={2}
  initTransform={() => transformInstance(0)}  // matrices at t=0
  updateTransform={(t) => transformInstance(t)} // matrices per frame
  initColor={() => colorInstance(0)}
  updateColor={(t) => colorInstance(t)}
/>
```

- `initTransform` / `initColor` set the starting matrices and colors; `updateTransform` / `updateColor` run every frame with the elapsed time.
- `animIdx` selects which embedded animation clip to play across all instances.

## Sprite animation & a character controller [(demo 69)](7.8-demo-sources.md#demospriteanimation)

A `<sprite>` with a tiled sheet becomes an animated 2D character. `DemoSpriteAnimation` slices an 8×8 sheet, wires WASD via `useKeyDown` / `useKeyUp`, and steps through frame indices per state (idle/run, left/right).

```jsx
const ref = useRefEffect((sprite) => {
  const map = sprite.material.map;    // 'images/sprites/character32px.png'
  map.magFilter = THREE.NearestFilter; // crisp pixels
  map.repeat.set(1 / 8, 1 / 8);        // one tile of the 8×8 sheet

  sprite.cc = { /* character controller: animations, keysPressed, moveSpeed… */ };
  sprite.cc.animations = {
    IDLE_RIGHT: { indices: [0, 1, 2, 3], maxTime: 1.25 / 4, index: 0, elapsedTime: 0 },
    RUN_RIGHT:  { indices: [8, 9, 10, 11, 12, 13, /*…*/], maxTime: 1.25 / 12, index: 0, elapsedTime: 0 },
    /* IDLE_LEFT, RUN_LEFT … */
  };

  useKeyDown((e) => (sprite.cc.keysPressed[e.key.toLowerCase()] = true));
  useKeyUp((e)   => (sprite.cc.keysPressed[e.key.toLowerCase()] = false));
});
```

Advancing an animation is just moving `map.offset` to the current tile each frame based on the active state's `indices`. `NearestFilter` keeps pixel art sharp.

## Spatial audio & visualizer [(demo 56)](7.8-demo-sources.md#demoaudio)

The `<audio>` element attaches a `THREE.AudioListener` and plays a buffer. Because browsers block autoplay, gate playback behind a click. An `AudioAnalyser` feeds a `DataTexture` you can visualize — rewritten as a TSL node material in v2.

```jsx
const ref = useRefEffect((audio) => {
  const button = makePlayButton();
  button.addEventListener('click', () => {
    loadAudios('sounds/terrainFS.mp3').then((buffer) => {
      audio.setBuffer(buffer);
      audio.play();
    });
  });
});

<audio ref={ref} args={[new THREE.AudioListener()]} setLoop={true} setVolume={0.2} />;
```

```jsx
// live FFT → texture → node material for a bar/waveform visualizer
const analyser = new THREE.AudioAnalyser(audio, 128);
const audioTex = new THREE.DataTexture(analyser.data, 64, 1, THREE.RedFormat);
const material = new THREE.MeshBasicNodeMaterial({ transparent: true, side: THREE.DoubleSide });
```

## Load, then save [(demo 70)](7.8-demo-sources.md#demoloadthensave)

`<Model>` round-trips: load a model, and later re-render it with `object={model}` plus an output `url` ending in `.glb` to export it. `DemoLoadThenSave` loads a `.zip`, waits, then re-emits it as `model.glb`.

```jsx
const ref = useRef(null);
const [model, setModel] = useState(null);

useEffect(() => {
  const id = setTimeout(() => setModel(ref.current), 1000);
  return () => clearTimeout(id);
}, []);

return model
  ? <Model url={'model.glb'} object={model} />              // save: serialize to model.glb
  : <Model ref={ref} url={'models/troll.zip'} onLoad={(m) => (ref.current = m)} />; // load
```

Supported inputs include `.glb`, `.gltf`, `.zip` (bundled gltf + textures), `.vox`, and MD2 archives — see [6.1](../advanced-react-elements/6.1-element-to-create-or-load-3d-models.md) for the full list.
