# 7.1 First render

This first group gets objects on the screen and moving. You will meet the four building blocks every later example relies on: the shortcut **`<box>`** element, the explicit **`<mesh>`** element, the **`<group>`** container, and **parent–child hierarchies**. Every snippet below runs inside `<DemoExamples />` (see [7. Examples](7.-examples.md)).

## A box, and three ways to animate it [(demo 1)](7.8-demo-sources.md#demobox)

`<box>` is a shortcut element: it creates a mesh from a `BoxGeometry` plus a material chosen by `type`. You never touch geometry or material objects directly.

```jsx
// Two boxes: a classic material (left) and its TSL node-material twin (right)
<box position={[-10, 0, 0]} scale={[15, 6, 12]} type={'phong'}     color={'red'} />
<box position={[ 10, 0, 0]} scale={[15, 6, 12]} type={'phongNode'} color={'yellow'} />
```

`type` selects the material: `'basic' | 'lambert' | 'phong' | 'standard' | 'physical' …`. The `…Node` variants (`'phongNode'`, `'standardNode'`, …) are the **TSL node materials**. Both boxes render on the **same** WebGPU renderer — under WebGPU even the classic materials are converted to nodes internally, so `type` does not pick a backend. Reach for a `…Node` variant only when you want to attach your own TSL nodes (`colorNode`, `positionNode`, …); otherwise the plain names are simpler.

The same box can be animated three different ways. They are equivalent — choose whichever fits your component.

```jsx
// (a) useFrame — a hook that runs every frame. Signature: (ct, threefy, dt)
const ref = useRef(null);
useFrame((ct, threefy, dt) => {
  const box = ref.current;
  if (!box) return;
  box.rotation.x += dt;
  box.scale.setScalar(8 * (1.0 + 0.5 * Math.sin(ct)));
  box.material.color.setHSL(0.5 + 0.2 * Math.sin(ct), 1, 0.5);
});
return <box ref={ref} type={'standard'} />;
```

```jsx
// (b) update — a per-object callback (dt, ct).
const update = function(dt, ct) {
  this.rotation.x += dt;
  this.scale.setScalar(8 * (1.0 + 0.5 * Math.sin(ct)));
};
return <mesh update={update}>
  <boxGeometry />
  <meshStandardMaterial />
</mesh>;
```

```jsx
// (c) onBeforeRender — three.js' native hook: (renderer, scene, camera, geometry, material)
function onBeforeRender(r, s, c, g, m) {
  this.rotation.x += 0.01;
  m.color.setHSL(Math.random(), 1, 0.5);
}
return <mesh onBeforeRender={onBeforeRender}>
  <boxGeometry />
  <meshStandardMaterial />
</mesh>;
```

- **`useFrame(ct, threefy, dt)`** — `ct` is elapsed time (seconds), `dt` is the delta since the last frame, `threefy` is `{ scene, camera, renderer, timer }`. Best when the animation needs component state or refs.
- **`update(dt, ct)`** — attached per element; `this` is the object3D. Best for simple, self-contained motion.
- **`onBeforeRender`** — three.js' own callback; use it when you need the renderer/scene/camera arguments.

## `<mesh>`: geometry + material spelled out [(demo 7)](7.8-demo-sources.md#demomesh)

`<mesh>` is the explicit form of a box. You provide a geometry child and a material child. This is what `<box>` expands to internally.

```jsx
<mesh
  position={[0, 0, 0]}
  rotation={[Math.PI / 4, 0, 0, 'XYZ']}
  scale={[20, 10, 10]}
  update={function (dt, ct) { this.rotation.x += Math.sin(dt) * 2.0; }}
>
  <boxGeometry args={[1, 1, 1]} />
  <meshStandardMaterial color="white" map="images/crate.gif" />
</mesh>
```

Geometries and materials can also be created once and shared across meshes — pass them through the `geometry` / `material` props instead of as children:

```jsx
const geometry = new THREE.BoxGeometry();
const material = new THREE.MeshStandardMaterial({ color: 'blue' });

<object3D scale={10}>
  <mesh position={[-1, -1, 0]}>
    <boxGeometry />
    <meshStandardMaterial color={'red'} />
  </mesh>
  <mesh position={[1, -1, 0]} geometry={geometry}>
    <meshStandardMaterial color={'green'} />
  </mesh>
  <mesh position={[-1, 1, 0]} material={material} >
    <boxGeometry />
  </mesh>
  <mesh position={[1, 1, 0]} geometry={geometry} material={material} />
</object3D>
```

Sharing one geometry/material instance across many meshes is the cheapest way to save memory when the meshes are identical. `<object3D>` is an empty container used purely to group and transform its children together.

## `<group>`: transform many objects as one [(demo 8)](7.8-demo-sources.md#demogroup)

A `<group>` applies its transform to every child. Give the group an `update` and the whole cluster moves; give each child its own `update` and they move independently on top of the group's motion.

```jsx
<group update={function (dt, ct) { this.rotation.z += 0.4 * dt; }}>
  <box update={function (dt, ct) { this.rotation.x += dt; }}
       position={[-8, -8, 0]} scale={[10, 10, 10]} type={'lambert'} color={'red'} />
  <box update={function (dt, ct) { this.rotation.y += dt; }}
       position={[8, -8, 0]} scale={[10, 10, 10]} type={'lambert'} color={'green'} />
  <box update={function (dt, ct) { this.scale.setScalar(10 + 3 * Math.sin(ct)); }}
       position={[8, 8, 0]} scale={[10, 10, 10]} type={'lambert'} color={0xffffaa}
       map={'images/crate.gif'} />
</group>
```

## Parent–child hierarchy [(demo 9)](7.8-demo-sources.md#demoboxorbit)

Nesting one element inside another makes the child inherit the parent's transform — the basis of orbits, arms, and articulated models. The two snippets below produce the identical result; the second just uses the `<box>` shortcut.

```jsx
function rotate(dt, ct) { this.rotation.z += dt; }

// explicit mesh form
<mesh update={rotate}>
  <boxGeometry /><meshStandardMaterial color={'red'} />
  <mesh update={rotate} position={[10, 0, 0]}>
    <boxGeometry /><meshStandardMaterial color={'green'} />
    <mesh position={[3, 0, 0]}>
      <boxGeometry /><meshStandardMaterial color={'yellow'} />
    </mesh>
  </mesh>
</mesh>
```

```jsx
// shortcut form — same scene, fewer lines
<box update={rotate} position={[0, 0, 0]} type={'standard'} color={'red'}>
  <box update={rotate} position={[10, 0, 0]} type={'standard'} color={'green'}>
    <box position={[3, 0, 0]} type={'standard'} color={'yellow'} />
  </box>
</box>
```

The green box orbits the red one; the yellow box rides along with the green one — each `position` is expressed **relative to its parent**.
