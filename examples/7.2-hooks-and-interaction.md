# 7.2 Hooks & interaction

Threefy ships a small set of React hooks that connect your components to the render loop, the input devices, the asset loaders, and the engine internals. This group covers the ones you reach for daily. See also [3. Hooks](../hooks/3.-hooks.md) and [4. Events](../events/4.-events.md).

## `useFrame`: run code every frame [(demo 4)](7.8-demo-sources.md#demouseframe)

`useFrame` registers a callback that fires once per rendered frame. Use it to drive animation from a component while keeping a `ref` to the object you mutate.

```jsx
const DemoUseFrame = (props) => {
  const ref = useRef(null);

  useFrame((ct, threefy, dt) => {
    // threefy = { scene, camera, renderer, timer, ... }
    if (ref.current) ref.current.rotation.x = ct * 1.5;
  });

  return (
    <mesh ref={ref} scale={10} {...props}>
      <boxGeometry />
      <meshStandardMaterial />
    </mesh>
  );
};
```

The callback receives `ct` (elapsed seconds), the `threefy` engine object, and `dt` (delta seconds). Prefer `dt`-based increments (`rotation.x += dt`) when you want frame-rate-independent motion, and `ct`-based expressions (`Math.sin(ct)`) for absolute, looping animation.

## `useKeyDown`: keyboard input [(demo 6)](7.8-demo-sources.md#demousekeydown)

`useKeyDown` runs a handler on every key-down event. Here it recolors a box in response to `r` / `g` / `b`.

```jsx
const DemoUseKeyDown = () => {
  const ref = useRefEffect((box) => {
    const color = box.material.color;
    useKeyDown((e) => {
      switch (e.key) {
        case 'r': color.set('red');   break;
        case 'g': color.set('green'); break;
        case 'b': color.set('blue');  break;
        default:  color.set('skyblue'); break;
      }
    });
  });

  return <box ref={ref} scale={10} type={'phong'} color={'skyblue'} />;
};
```

`useRefEffect` gives you the object the moment it is created — a convenient place to wire up an input handler that closes over `box.material.color`.

## Pointer events [(demo 20)](7.8-demo-sources.md#demoevents)

Any element accepts pointer and mouse event props. Each handler is called with the DOM-style event and the object that was hit.

```jsx
<mesh scale={10}
  onClick={(e, mesh)        => console.log('onClick', mesh)}
  onDoubleClick={(e, mesh)  => console.log('onDoubleClick')}
  onContextMenu={(e, mesh)  => console.log('onContextMenu')}
  onWheel={(e, mesh)        => console.log('onWheel')}
  onPointerMove={(e, mesh)  => console.log('onPointerMove')}
  onPointerEnter={(e, mesh) => console.log('onPointerEnter')}
  onPointerLeave={(e, mesh) => console.log('onPointerLeave')}
  onPointerDown={(e, mesh)  => console.log('onPointerDown')}
  onPointerUp={(e, mesh)    => console.log('onPointerUp')}
>
  <boxGeometry />
  <meshStandardMaterial color={'red'} />
</mesh>
```

Threefy raycasts against the scene for you — no manual `Raycaster` setup is needed. `onPointerEnter` / `onPointerLeave` fire on hover boundaries, making highlight-on-hover trivial.

## `useThree`: read and mutate the engine [(demo 21)](7.8-demo-sources.md#demousethree)

`useThree()` exposes three helpers over the live engine — `get`, `set`, and `list`. It is the escape hatch for anything the declarative elements don't cover.

```jsx
const { get, set, list } = useThree();

// get(target): read an engine object, a nested property, or a bound method
const scene    = get('scene');
const camera   = get('camera');
const dpr      = get('renderer.getPixelRatio')();
const [w, h]   = get('width', 'height');           // multiple gets at once

// set(target, value): mutate the engine. `target` starts with one of
// scene | camera | renderer | timer | controls | animator | raycaster
set('renderer.toneMapping', THREE.ACESFilmicToneMapping);
set('scene.background', 'images/brickWall/diffuse.jpg');   // string paths are auto-loaded

// set also accepts an object of updates, or a "call this method" map:
set(get('renderer'), {
  setPixelRatio: 0.5,                  // → renderer.setPixelRatio(0.5)
  setViewport: [100, 100, 700, 700],   // → renderer.setViewport(...)
});

// list(target): enumerate the members of an engine object
list(get('scene'));
```

You can even swap the camera or scene wholesale:

```jsx
const ortho = new THREE.OrthographicCamera(-25, 25, 25, -25, 0.1, 1000);
ortho.position.z = 500;
set('camera', ortho);
```

Everything here runs against the same engine the declarative tree renders into, so a `set('scene.background', …)` in a `useEffect` is reflected on the next frame.

## `useLoader`: load models and textures [(demo 25)](7.8-demo-sources.md#demouseloader)

`useLoader` (and the `<Model>` element) resolve asset URLs into ready-to-use three.js objects, integrating with React Suspense.

```jsx
// Textures — pass an array of URLs, get back loaded THREE.Texture objects
const [tex1, tex2, tex3] = useLoader([
  'images/crate.gif',
  'images/brickWall/diffuse.jpg',
  'images/brickWall/normal.jpg',
]);
tex2.colorSpace = THREE.SRGBColorSpace;

const m = new THREE.MeshStandardMaterial({ map: tex2, normalMap: tex3 });
```

```jsx
// Models — <Model> loads .glb / .gltf / .zip and calls onLoad with the object
const onLoad = (model) => {
  if (model.animations) {
    const { animator } = useThree();
    animator.playAction(model, model.animations[0]);
  }
};

<group scale={10}>
  <Model url={'models/jump-animation.glb'} onLoad={onLoad} />
  <Model url={'models/troll.zip'} />
</group>;
```

`onLoad` is the place to fit the camera, start an animation clip via the `animator`, or apply a one-time transform. Model loading is covered in depth in [7.6 Models & assets](7.6-models-and-assets.md).

## `useAnimate`: pause / replay the loop on interaction [(demo 30)](7.8-demo-sources.md#demouseanimate)

`useAnimate` controls the render loop itself — handy for interaction-driven scenes that should only redraw when something changes. It returns `replay`, `pause`, and `flush`.

```jsx
const DemoUseAnimate = () => {
  const ref = useRef(null);
  const [hovered, setHovered] = useState(false);
  const [scale, setScale] = useState(1);
  const { replay, pause, flush } = useAnimate();

  useFrame((t) => { if (ref.current) ref.current.rotation.y = t; });

  return (
    <mesh
      ref={ref}
      scale={scale}
      onClick={() => { setScale(scale === 1 ? 1.5 : 1); flush(); }}
      onPointerOver={() => { setHovered(true);  pause();  }}
      onPointerOut={()  => { setHovered(false); replay(); }}
    >
      <boxGeometry args={[7, 15, 9]} />
      <meshStandardMaterial color={hovered ? 'green' : 'white'}
        map={'images/brickWall/diffuse.jpg'} />
    </mesh>
  );
};
```

- **`pause()` / `replay()`** — stop and resume the animation loop (here, freeze rotation while hovered).
- **`flush()`** — force a single re-render after a React state change so the new `scale` is applied immediately.
