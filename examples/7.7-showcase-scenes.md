# 7.7 Showcase scenes

The final group combines everything from the earlier pages into complete little worlds — animated water, terrain under a physical sky, procedural curves, and motion along a path. These lean on Threefy's higher-level nature elements (`<Water>`, `<Sky>`, `<Terrain>`) and, in v2, TSL node materials. Several were rewritten for WebGPU; the notes call out how. See also [6.4 Elements to support nature modeling](../advanced-react-elements/6.4-elements-to-support-nature-modeling.md).

## Ocean scene [(demo 35)](7.8-demo-sources.md#demooceanscene)

`<Water>` is a self-animating water surface. In v2 the `WaterMesh` drives its ripples from a TSL `time` node, so you **don't** update a time uniform in `useFrame` anymore — you only animate your own objects.

```jsx
<scene>
  <background url={'images/hdr/table_mountain_puresky_4k.hdr'} />
  <hemisphereLight args={[0x7cbfff, 0xffe5bc, 0.2 * Math.PI]} position={[0, 1, 0]} />
  <box ref={boxRef} scale={10} type={'standard'} map={'images/crate.gif'} />
  <Water ref={waterRef} />
</scene>
```

```jsx
useFrame((t) => {
  const box = boxRef.current;
  if (!box) return;
  box.position.y = Math.sin(t) * 7 + 2;   // bob on the waves
  box.rotation.x = t * 0.5;
});
```

An HDR `<background>` doubles as image-based lighting, so the box picks up realistic reflections from the sky.

## A ship rocking on the sea [(demo 36)](7.8-demo-sources.md#demoshiponthesea)

Same `<Water>`, but now a loaded ship model pitches, yaws, and rolls with three out-of-phase sine waves — a convincing "floating" motion built from nothing but trig.

```jsx
const ship = useLoader('models/blackPearl.zip');

useFrame((t) => {
  const s = shipRef.current;
  if (!s) return;
  s.position.y = Math.sin(t) * 0.4 + 2.5;
  s.rotation.x = Math.PI / 64 * Math.sin(t * 0.5); // pitch
  s.rotation.y = Math.PI / 64 * Math.sin(t * 0.1); // yaw
  s.rotation.z = Math.PI / 24 * Math.sin(t * 1.0); // roll
});

<scene>
  <background url={'images/hdr/overcast_soil_puresky_4k.hdr'} />
  <hemisphereLight args={[0x7cbfff, 0xffe5bc, 0.2 * Math.PI]} />
  <primitive ref={shipRef} object={ship} scale={0.1} onLoad={onLoad} />
  <Water ref={waterRef} />
</scene>;
```

Using **different frequencies** for pitch/yaw/roll keeps the motion from looking mechanically periodic.

## Sky + terrain with image-based lighting [(demo 37)](7.8-demo-sources.md#demoskyterrain)

`<Sky>` renders a physical sky whose sun position you set from azimuth/elevation, and `<Terrain>` builds a displaced landscape from a diffuse + height map. The sky is also baked into `scene.environment` so everything is lit by it.

```jsx
const skyRef = useRefEffect((sky) => {
  const { threefy, scene, renderer } = useThree();

  // sun direction from (elevation, azimuth)
  const phi = THREE.MathUtils.degToRad(90 - 2);    // elevation 2°
  const theta = THREE.MathUtils.degToRad(180);     // azimuth 180° (south)
  sky.sunPosition.value.copy(
    new THREE.Vector3().setFromSphericalCoords(1, phi, theta)); // v2: uniforms as properties

  // bake the sky into scene.environment — WebGPU backend init is async,
  // so wait for threefy.ready before PMREMGenerator.fromScene()
  threefy.ready.then(() => {
    const env = new THREE.Scene();
    env.add(sky);
    scene.environment = new THREE.PMREMGenerator(renderer).fromScene(env).texture;
    scene.add(sky);
  });
});

<scene>
  <Sky ref={skyRef} />
  <Terrain position-y={-100} xsize={500} ysize={300} zsize={500}
    map={'images/terrain/Rugged Terrain/diffuse.jpg'}
    heightMap={'images/terrain/Rugged Terrain/heightmap.jpg'} />
</scene>;
```

⚠ v2 note: WebGPU initializes the renderer backend asynchronously. Anything that renders to a target at startup (like `PMREMGenerator.fromScene`) must wait for `threefy.ready`.

## Heightmap terrain with a TSL displacement material [(demo 46)](7.8-demo-sources.md#demoheightmapterrain)

This is the "roll your own terrain" version: a plane whose vertices are pushed up by a heightmap. In v1 that used GLSL injection into a custom material; **v2 rewrites it as a TSL `positionNode`** because WebGPU can't take injected GLSL.

```jsx
import { positionLocal, normalLocal, texture, uv } from 'three/tsl';

// position = position + normal * heightmap.r * height
const positionNode = positionLocal.add(
  normalLocal.mul(texture(heightTex, uv()).x).mul(70)
);

// the concise "type" form — a plane driven by a node material
<plane
  rotation-x={-Math.PI / 2}
  position-y={-20}
  args={[100, 100, 64, 64]}
  type={'standardNode'}
  map={diffuseUrl}
  normalMap={normalUrl}
  positionNode={positionNode}
/>;
```

The same `positionNode` works three ways: passed to a hand-built `MeshStandardNodeMaterial`, to a `<meshStandardNodeMaterial>` child, or to the `type={'standardNode'}` shortcut shown here. A denser plane (`64×64` segments) gives the displacement more vertices to work with.

## Procedural curves: a spirograph [(demo 64)](7.8-demo-sources.md#demospirograph)

`<curve>` turns a math function into drawable geometry. `DemoSpirograph` traces the classic `r1·cos(t) + r2·cos(t·ratio)` pattern, either as one smooth Catmull-Rom curve (`monochrome`) or as many colored segments (`polychrome`).

```jsx
// build the point list from the spirograph equations
const points = [];
for (let t = 0; t <= tmax; t += dt) {
  points.push([
    r1 * Math.cos(t) + r2 * Math.cos(t * ratio),
    r1 * Math.sin(t) + r2 * Math.sin(t * ratio),
    0,
  ]);
}

<curve type={'catmullRom3'} args={[points]} divisions={points.length - 1} />;
```

Different `r1 / r2 / ratio` values produce completely different rosettes — try `r1=5, r2=5, ratio=3.14` for a dense flower.

## Moving an object along a path [(demo 66)](7.8-demo-sources.md#demomovealongpath)

Build a `<curvePath>` from line and bezier segments, then sample it every frame to drive a model's position **and orientation** — the model turns to face its direction of travel.

```jsx
const MoveAlongPath = ({ children, ...props }) => {
  const up = new THREE.Vector3(0, 1, 0);
  let t = 0;

  const ref = useRefEffect((group) => {
    const model = group.children[0];
    const path = group.children[1].userData['curve'] || group.children[1].userData['path'];

    useFrame(() => {
      model.position.copy(path.getPoint(t));               // where on the path
      const tangent = path.getTangent(t);                  // which way it points
      const axis = new THREE.Vector3().crossVectors(up, tangent).normalize();
      model.quaternion.setFromAxisAngle(axis, Math.acos(up.dot(tangent)));
      t = (t + 0.002) % 1;                                 // loop 0→1
    });
  });

  return <group ref={ref} {...props}>{children}</group>;
};

<MoveAlongPath>
  <Mesh>
    <cylinderGeometry args={[0.4, 0.6, 3, 10]} />
    <coneGeometry args={[1, 2, 10]} translate={[0, 2.5, 0]} />
    <material type={'standard'} map={'images/raymarch/abstract1.jpg'} />
  </Mesh>
  <curvePath>
    <curve type={'line'}   dim={3} color={'red'}   args={[[0, 0, 0], [-10, 0, 0]]} />
    <curve type={'line'}   dim={3} color={'green'} args={[[-10, 0, 0], [-10, 10, 0]]} />
    <curve type={'bezier'} order={'cubic'} dim={3} color={'yellow'} divisions={20}
           args={[[0, 10, 0], [20, 10, 0], [20, 0, 0], [0, 0, 0]]} />
  </curvePath>
</MoveAlongPath>;
```

`getPoint(t)` and `getTangent(t)` (with `t` from 0 to 1) are all you need: position from the point, facing from the tangent. Wrapping `t` back to 0 loops the motion forever.
