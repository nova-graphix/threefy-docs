# 7.4 Geometry & materials

Every visible object is a geometry (its shape) plus a material (its surface). This group walks from the built-in geometry catalog down to raw vertex buffers, then from ready-made materials up to physically based rendering and custom TSL node materials. See also [5.1 Core](../basic-react-elements/5.1-core.md).

## The geometry catalog [(demo 33)](7.8-demo-sources.md#demogeometries)

Threefy exposes every three.js geometry as a lowercase element. `DemoGeometries` lays out the whole set on a grid — box, sphere, cone, torus, torusKnot, capsule, lathe, tube, the platonic solids, plus `shape`, `extrude`, `polyhedron`, and the line helpers `edgesGeometry` / `wireframeGeometry`.

```jsx
<mesh><boxGeometry /><meshStandardMaterial map={map} /></mesh>
<mesh><sphereGeometry args={[0.75]} /><meshStandardMaterial map={map} /></mesh>
<mesh><torusKnotGeometry args={[0.5, 0.2]} /><meshStandardMaterial map={map} /></mesh>
<mesh><capsuleGeometry args={[0.5, 0.5]} /><meshStandardMaterial map={map} /></mesh>

// custom polyhedron from vertices + faces
<mesh>
  <polyhedronGeometry args={[verticesOfCube, indicesOfFaces, radius, detail]} />
  <meshStandardMaterial map={map} />
</mesh>

// edges / wireframe overlays are line geometries
<lineSegments>
  <edgesGeometry args={[sphereGeometry]} />
  <lineBasicMaterial color={0xaaaaaa} />
</lineSegments>
```

`args` are forwarded straight to the three.js constructor, so any signature from the three.js docs works. Flat shapes (`plane`, `circle`, `ring`, `shape`) usually want `side={THREE.DoubleSide}` so they render from both sides.

## Custom geometry from a buffer [(demo 18)](7.8-demo-sources.md#demobuffergeometry)

When no built-in shape fits, build a `<bufferGeometry>` from typed arrays and attach attributes. Here 16 000 randomly placed triangles get positions, flat normals, and per-vertex RGBA colors.

```jsx
const { positions, normals, colors } = randomTriangles(); // Float32Arrays

<mesh update={rotate}>
  <bufferGeometry>
    <bufferAttribute attach="attributes-position" args={[positions, 3]} />
    <bufferAttribute attach="attributes-normal"   args={[normals, 3]} />
    <bufferAttribute attach="attributes-color"    args={[colors, 4]} />
  </bufferGeometry>
  <meshStandardMaterial color={0xd5d5d5} side={THREE.DoubleSide} vertexColors transparent />
</mesh>
```

- `attach="attributes-<name>"` binds an attribute onto the geometry.
- `args={[array, itemSize]}` — `itemSize` is components per vertex (3 for position/normal, 4 for RGBA color).
- `vertexColors` on the material tells it to read the `color` attribute.

## Shapes: 2D outlines to 3D solids [(demo 54)](7.8-demo-sources.md#demoshapes)

A `THREE.Shape` is a 2D outline (with optional holes) that you turn into a flat `shapeGeometry` or an `extrudeGeometry` solid. `DemoShapes` builds a fish (with a circular hole) and a heart.

```jsx
const fishShape = new THREE.Shape()
  .moveTo(x, y)
  .quadraticCurveTo(x + 50, y - 80, x + 90, y - 10)
  /* … */
  .quadraticCurveTo(x + 50, y + 80, x, y);
fishShape.holes.push(new THREE.Path().absarc(x + 20, y, 10, 0, Math.PI * 2, true));

const extrudeSettings = { depth: 8, bevelEnabled: true, bevelSize: 1, bevelThickness: 1, steps: 2 };

// flat outline
<mesh><geometry type={'shape'} args={[fishShape, 12]} /><material type={'phong'} side={THREE.DoubleSide} /></mesh>

// extruded solid — same shape given depth + bevel
<mesh><geometry type={'extrude'} args={[fishShape, extrudeSettings]} /><material type={'phong'} side={THREE.DoubleSide} /></mesh>
```

`<geometry type={…}>` / `<material type={…}>` are the generic forms — one element that becomes any geometry/material by name, handy when the type is data-driven. There is also a `<shape>` shortcut element that fuses the two.

## Curves & lines [(demo 53)](7.8-demo-sources.md#democurves)

Curves (`Path`, bezier and spline curves, `CatmullRomCurve3`) produce point lists you render as lines. The pipeline is: build a curve → `getPoints()` → feed a buffer geometry → draw with a line material.

```jsx
const path = new THREE.Path();
path.moveTo(0.5, 0.3);
path.quadraticCurveTo(0.5, 0.5, 0.3, 0.5);
path.lineTo(-0.3, 0.5);
/* … */
const points = path.getPoints();

const ref = useRefEffect(() => ref.current.computeLineDistances()); // needed for dashes

<line ref={ref} scale={10}>
  <geometry type={'buffer'} setFromPoints={[points]} />
  <material type={'dashed'} color={0xffff00} dashSize={0.05} gapSize={0.05} />
</line>
```

- `setFromPoints={[points]}` fills the buffer geometry directly from a point array. Points can be `Vector3`s, `[x,y,z]` arrays, flat number arrays, or `{x,y,z}` objects.
- `LineDashedMaterial` needs `computeLineDistances()` after the geometry is set.
- Note: `linewidth` is always 1 with the standard line material — for thick lines use fat-line addons.

## PBR materials & texture sub-props [(demo 49)](7.8-demo-sources.md#demopbrmaterials)

For realistic surfaces, feed a `standard` (or `physical`) material a full set of PBR maps. Threefy adds concise **texture sub-prop** syntax so you can tile and transform every map without touching `THREE.Texture` objects.

```jsx
<sphere args={[16, 64, 32]} type={'standard'}
  textureRepeat={2}                          // tiles ALL maps 2×2 at once
  textureRotation={Math.PI / 2}              // rotate all maps (radians)
  textureCenter={0.5}                        // rotate about the tile center
  map={'images/rockMossy/diffuse.jpg'}
  normalMap={'images/rockMossy/normal.jpg'}
  aoMap={'images/rockMossy/ao.jpg'}
  roughnessMap={'images/rockMossy/aoRoughMetal.jpg'}
  displacementMap={'images/rockMossy/displacement.jpg'}
  displacementScale={2}
/>
```

- `textureRepeat` / `textureRotation` / `textureCenter` apply to **every** map at once — the recommended PBR shortcut.
- Per-map control uses a hyphenated sub-prop: `map-repeat={2}`, `normalMap-repeat={[2, 2]}`, `map-colorSpace={'srgb'}`.
- ⚠ Do not set a global `textureColorSpace` — it would wrongly push data maps (normal/roughness) into sRGB. Color-space is a per-map concern; the loader already tags color maps as sRGB.

## Custom node materials with TSL [(demo 45)](7.8-demo-sources.md#demoshadermaterial)

In v2 (WebGPU), custom shaders are written with **TSL** (Three Shading Language) node materials instead of GLSL strings. `DemoShaderMaterial` deforms a box along a sine wave and picks one of four textures by x-position — the `time` node updates automatically, so no `useFrame` uniform juggling is needed.

```jsx
import { vec3, vec4, sin, time, texture, uv, positionLocal } from 'three/tsl';

const material = new THREE.MeshBasicNodeMaterial();

// vertex: y += sin(x/3 + time)
material.positionNode = vec3(
  positionLocal.x,
  sin(positionLocal.x.div(3).add(time)).add(positionLocal.y),
  positionLocal.z
);

// fragment: choose a texture by the original x coordinate
material.colorNode =
  positionLocal.x.greaterThan(10).select(texture(maps[0], uv()),
  positionLocal.x.greaterThan(0).select(texture(maps[1], uv()),
  positionLocal.x.greaterThan(-10).select(texture(maps[2], uv()),
  texture(maps[3], uv()))));

return <primitive object={new THREE.Mesh(geometry, material)} />;
```

- `positionNode` replaces the vertex position; `colorNode` replaces the fragment color.
- `time` is a built-in node that advances every frame on its own.
- `.greaterThan(…).select(a, b)` is TSL's branch-free conditional — the node graph compiles to WGSL for WebGPU.
