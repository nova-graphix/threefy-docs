# 7.3 Many objects & performance

Drawing thousands of objects as individual meshes is slow — every mesh is a separate draw call. Threefy exposes the four three.js techniques that cut that cost, and the right one depends on whether your objects share geometry and material. This group teaches the decision and each element. See also [6.2 Elements for mesh, instanced mesh and merged mesh](../advanced-react-elements/6.2-elements-for-mesh-instanced-mesh-and-merged-mesh.md).

## Choosing a technique [(demo 44)](7.8-demo-sources.md#democomparemeshes)

`DemoCompareMeshes` puts all four side by side. Use this table to pick:

| Technique | Geometry | Material | Best for |
| --- | --- | --- | --- |
| plain `<mesh>` | different | different | a handful of unique objects |
| `<instancedMesh>` | **same** | **same** | thousands of identical objects (trees, cubes) |
| `<batchedMesh>` | different | same (one material, per-instance values) | many varied shapes, still animated individually |
| `<MergedMesh>` | different | same | many static shapes baked into **one** draw call |

The rest of this page shows each in turn.

## `<instancedMesh>`: same geometry, same material [(demo 11)](7.8-demo-sources.md#demoinstmesh)

One geometry and one material are uploaded once; each instance only carries a transform matrix and (optionally) a color. This is the cheapest way to draw thousands of identical objects.

```jsx
const DemoInstMesh = () => {
  const instCount = 1000;

  const ref = useRefEffect((instMesh) => {
    for (let i = 0; i < instCount; i++) {
      instMesh.setMatrixAt(i, randomMatrix(40, 40, 40, 0.5, 1));
      instMesh.setColorAt(i, randomHSLColor());
    }
  });

  useFrame((t) => {
    if (ref.current) {
      ref.current.rotation.x = Math.sin(t * 1.2);
      ref.current.rotation.y = Math.sin(t * 2.4);
    }
  });

  return (
    <instancedMesh ref={ref} count={instCount}>
      <boxGeometry />
      <meshStandardMaterial />
    </instancedMesh>
  );
};
```

- `count` sizes the instance buffers up front.
- `setMatrixAt(i, matrix)` places instance `i`; `setColorAt(i, color)` tints it.
- Set these once in `useRefEffect` (creation) or every frame in `useFrame` (animation). When you change matrices at runtime, set `instMesh.instanceMatrix.needsUpdate = true`.

## `<batchedMesh>`: different geometries, one material [(demo 14)](7.8-demo-sources.md#demobatchedmesh)

A batched mesh holds several distinct geometries but a single material, and still lets you drive each instance's matrix and per-instance material values (roughness, metalness, diffuse, emissive) every frame.

```jsx
const geometries = [
  new THREE.ConeGeometry(1.0, 2.0),
  new THREE.BoxGeometry(2.0, 2.0, 2.0),
  new THREE.SphereGeometry(1.0, 16, 8),
];
const amount = 512;
const ids = [];

const ref = useRefEffect((batchMesh) => {
  const geomIds = geometries.map((g) => batchMesh.addGeometry(g));
  for (let i = 0; i < amount; i++) {
    const id = batchMesh.addInstance(geomIds[i % geomIds.length]);
    ids.push(id);
    batchMesh.setMatrixAt(id, randomMatrix(40, 40, 40, 0.5, 1));
    batchMesh.material.setValue(id, 'roughness', 0);
  }
});

return (
  <batchedMesh ref={ref} args={[amount, amount * 512, amount * 1024]}>
    <batchedMaterial args={[amount]} map={'images/crate.gif'} />
  </batchedMesh>
);
```

- `args={[maxInstanceCount, maxVertexCount, maxIndexCount]}` reserves the buffers.
- `addGeometry(geo)` registers a geometry and returns its id; `addInstance(geomId)` spawns an instance of it.
- `<batchedMaterial>` enables per-instance material values through `material.setValue(id, key, value)` — animate `diffuse`, `emissive`, `roughness`, `metalness` per instance in `useFrame`.

## `<MergedMesh>`: bake many statics into one [(demo 41)](7.8-demo-sources.md#demomergedmesh)

When objects never change individually, merge their geometries into a single mesh — one draw call, zero per-object overhead. `mergeMeshes()` does the baking; the `<MergedMesh>` element does it declaratively for its children.

```jsx
// Imperative: merge the meshes of a loaded model
const onLoad = (model) => {
  const meshes = [];
  model.traverse((o) => o.isMesh && meshes.push(o));
  const withUV = meshes.filter((m) => m.geometry.hasAttribute('uv'));

  mergeMeshes(withUV).then((merged) => {
    const parent = model.parent;
    parent.remove(model);
    parent.add(merged);
  });
};

return <Model url={'models/sponza.zip'} onLoad={onLoad} />;
```

```jsx
// Declarative: wrap children in <MergedMesh>
<MergedMesh position={[-25, 0, 0]}>
  {gridCoords(10, 5, 3).map((p, i) => (
    <box key={i} position={p} type={'standard'} map={someTexture} />
  ))}
</MergedMesh>
```

Merging is for **static** geometry — once baked, individual children can no longer be transformed independently. Note the meshes must be mergeable (compatible attributes, e.g. all have `uv`).

## `<lod>`: swap detail by distance [(demo 57)](7.8-demo-sources.md#demolod)

Level-of-detail renders a high-poly mesh up close and progressively cheaper meshes farther away. Threefy offers three equivalent ways to register the levels.

```jsx
// (a) imperative — addLevel(object, distance)
const ref = useRefEffect((lod) => {
  const mat = new THREE.MeshBasicMaterial({ wireframe: true });
  lod.addLevel(new THREE.Mesh(new THREE.IcosahedronGeometry(10, 3), mat), 0);
  lod.addLevel(new THREE.Mesh(new THREE.IcosahedronGeometry(10, 2), mat), 75);
  lod.addLevel(new THREE.Mesh(new THREE.IcosahedronGeometry(10, 1), mat), 150);
});
return <lod ref={ref} />;
```

```jsx
// (b) declarative — children become the levels, distances via addLevels prop
<lod addLevels={[[level0, 0], [level1, 75], [level2, 150]]} />
```

Each pair is `[object, distance]`: the object is shown once the camera is at least `distance` away. Combine LOD with instancing for large populated scenes.
