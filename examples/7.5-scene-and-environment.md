# 7.5 Scene & environment

These examples set the mood of a scene rather than its objects: shadows, camera controls, fog, backgrounds, reflections, and full-frame post effects. Several of them were rewritten for the v2 WebGPU pipeline — the notes below flag what changed.

## Shadows [(demo 22)](7.8-demo-sources.md#demoshadow)

Shadows need three things: a light with `castShadow`, objects that `castShadow`, and a surface that `receiveShadow`. The light's shadow camera frustum must enclose the shadow-casting objects.

```jsx
<directionalLight
  castShadow
  position={[10, 30, 0]}
  shadow-mapSize={[512, 512]}
  shadow-camera-left={-20}   shadow-camera-right={20}
  shadow-camera-top={20}     shadow-camera-bottom={-20}
  shadow-camera-near={0.5}   shadow-camera-far={70}
/>

<box castShadow    position={[-10, 15, 0]} scale={5} type={'lambert'} color={'red'} />
<box receiveShadow position={[0, 0, 0]} scale={[80, 2, 80]} type={'lambert'} color={'white'} />
```

The hyphenated props (`shadow-mapSize`, `shadow-camera-left`, …) drill into the light's nested shadow object. A bigger `shadow-mapSize` sharpens the shadow at the cost of memory; a tighter frustum sharpens it for free.

## Camera controls [(demo 50)](7.8-demo-sources.md#democontrols)

Threefy wraps every three.js control as an element. Drop one into the scene and pass the standard `[camera, renderer.domElement]` (some need the `scene` or `document.body` too).

```jsx
const { camera, renderer, scene } = useThree();

// the everyday choice
<orbitControls args={[camera, renderer.domElement]} enableDamping dampingFactor={0.075} />
```

The full set: `orbitControls`, `mapControls`, `trackballControls`, `arcballControls`, `dragControls`, `transformControls`, `firstPersonControls`, `flyControls`, `pointerLockControls`. `dragControls` takes `[scene.children, camera, domElement]` to move objects; `transformControls` adds an on-screen gizmo. When no control is present, Threefy installs a damped `OrbitControls` by default.

## Fog [(demo 51)](7.8-demo-sources.md#demofog)

Fog fades distant geometry into a color — cheap depth cueing. Two kinds: linear `<fog>` (near/far) and exponential `<fogExp2>` (density).

```jsx
// linear: fully clear at `near`, fully fogged at `far`
<fog args={[0xcccccc, 1, 1024]} />

// exponential: thickness grows with distance
<fogExp2 args={[0xcccccc, 0.002]} />
```

Fog is a scene property, so a single `<fog>` element anywhere in the tree affects everything. Match the fog color to your background for a seamless horizon.

## Backgrounds [(demo 59)](7.8-demo-sources.md#demobackground)

The `<background>` element sets `scene.background` from a color, a single image, a six-image cubemap, or an HDR file.

```jsx
<background color={'blue'} />                         // solid color
<background url={'images/background/leather.jpg'} />  // equirectangular / flat image
<background url={[                                    // cubemap: +x −x +y −y +z −z
  'cloud/px.png', 'cloud/nx.png',
  'cloud/py.png', 'cloud/ny.png',
  'cloud/pz.png', 'cloud/nz.png',
]} />
<background url={'images/hdr/landscape.hdr'} />       // HDR environment
```

A cubemap or HDR background can double as the environment map for reflective materials.

## Reflection & refraction with an env map [(demo 61)](7.8-demo-sources.md#democuberefraction)

`DemoCubeRefraction` loads a cubemap background, then reuses it as the `envMap` of a `phong` material with a `refractionRatio` — the mesh refracts the surroundings like glass.

```jsx
// 1) set a cubemap background and grab it once loaded
const { scene, threefy } = useThree()
scene.background = threefy.loadTexture('images/background/leather.jpg')
scene.background.mapping = THREE.CubeRefractionMapping
const envTex = scene.background

// 2) feed it to the material as an env map
return <mesh>
  <primitive object={new THREE.BoxGeometry(10,10,10)} attach={'geometry'} />
  <material type={'phong'} color={0xffffff} envMap={envTex} refractionRatio={0.98} />
</mesh>
```

`refractionRatio` near 1.0 bends light slightly (thin glass); lower values bend it more. Swap the mapping to `THREE.CubeReflectionMapping` for a mirror finish instead.

## Post-processing passes [(demo 52)](7.8-demo-sources.md#demopostprocessing)

Post-processing runs full-frame effects after the scene renders. Add pass elements and Threefy assembles the pipeline. **In v2 these are TSL-based** — several GLSL passes were rewritten and no longer need manual uniforms.

```jsx
// TSL passes — resolution is tracked automatically
<dotScreenPass scale={3} />
<rGBShiftPass amount={0.0015} />
<fXAAPass />          // was FXAAShader; no uniform setup needed
<glitchPass />

// ground-truth ambient occlusion with denoise
<gTAOPass
  args={[scene, camera, width, height]}
  output={GTAOPass.OUTPUT.Denoise}
  updateGtaoMaterial={{ radius: 0.25, thickness: 2, scale: 3, samples: 16 }}
/>
```

Passes render in the order you declare them. Keep anti-aliasing (`fXAAPass`) last so it smooths the final composited image.

## Camera / stereo effects [(demo 55)](7.8-demo-sources.md#democameraeffects)

Effects wrap the *renderer* rather than inserting a pass — anaglyph 3D, stereo/VR split, parallax barrier, and outline.

```jsx
const renderer = useThree().get('renderer');

<stereoEffect args={[renderer]} />                 // side-by-side stereo
<anaglyphEffect args={[renderer, width, height]} /> // red/cyan 3D glasses
<outlineEffect args={[renderer, { defaultThickness: 0.003, defaultColor: [1,1,1] }]} />
```

⚠ v2 note: `asciiEffect` is **not supported** on the WebGPU pipeline. The others work as shown.
